Kalkulator Energi LoRa

Estimasi umur baterai node ESP32 dan SX1276 untuk skenario monitoring IoT.

Kalkulator Panduan

Input Parameter

Gunakan parameter sesuai spesifikasi perangkat dan kondisi implementasi agar estimasi lebih akurat.

? Masukkan kapasitas nominal baterai/powerbank. Rentang umum perangkat lapangan: 2000 sampai 20000 mAh. Nilai contoh: 5000 mAh.
? Baterai Lithium umumnya menggunakan tegangan nominal 3.7V. Nilai ini adalah standar sel Li-ion/Li-po.
Rekomendasi teknis: 3.7V (sel Lithium).
? Saat tidak transmit LoRa, ESP32 umumnya berada di 40-50 mA. Saat sistem berjalan aktif (sensor, komputasi, I/O), arus sering naik ke 60-90 mA.
Referensi lapangan: idle 40-50 mA, aktif 60-90 mA.
? SX1276 pada daya pancar 20 dBm umumnya sekitar 120 mA. Kisaran implementasi umum: 105-130 mA tergantung modul dan efisiensi catu daya.
Referensi 20 dBm: 120 mA (range 105-130 mA).
? Referensi BW 125 kHz payload pendek: SF7 62 ms, SF8 113 ms, SF9 205 ms, SF10 371 ms, SF11 741 ms, SF12 1319 ms. Nilai aktual berubah sesuai payload dan coding rate.
Pilih SF dari dropdown atau gunakan Custom untuk isi airtime manual.
? Sesuaikan dengan variabel interval pada firmware. Jika TX_INTERVAL 10, maka perangkat mengirim setiap 10 detik. Interval lebih rapat akan meningkatkan konsumsi energi per jam.

Input Data Lapangan (OLED)

? Masukkan nilai energi total yang tampil pada OLED perangkat. Nilai ini digunakan untuk estimasi sisa kapasitas baterai aktual.
Masukkan angka total dari OLED ESP32 (bukan per paket).
